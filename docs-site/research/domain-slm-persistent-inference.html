1. Executive Summary

Baker's three-tier architecture (Render → Mac Mini → MacBook) is architecturally ahead of most production agent systems in 2026. The question was: can a domain-trained small language model replace cloud APIs for signal classification?

Bottom line: Fine-tune Qwen 3.5 4B on 2,000 curated classification examples from Baker's email/WhatsApp history. Serve via Ollama alongside Gemma 4. Wire into classify_intent(). Total investment: 2-3 days of work, zero ongoing cost.
Key Findings:
  1. Fine-tuning is feasible on Mac Mini M4 Pro (24GB). MLX + QLoRA can fine-tune a 7B model in 30 minutes. Full pipeline: MLX → fuse → GGUF → Ollama. Timeline: 2-3 days.
  2. Recommended base model: Qwen 3.5 4B or Gemma 4 E4B. Both fit on 24GB, run at 50-80 tok/s, co-exist with Gemma 4 12B.
  3. The case is latency and independence, not cost. Cloud classification costs <$1/month. Local is 5-10x faster (50ms vs 300ms) and works offline.
  4. NVIDIA patterns are architecture references, not tools. Nemotron fine-tuning requires NVIDIA GPUs. But AI-Q's routing patterns are directly adoptable.
  5. Baker's persistent inference is already best-in-class. cortex_events matches Anthropic Managed Agents. Don't adopt frameworks — adopt the 3-4 missing patterns.
  6. Karpathy's LLM Wiki IS Baker Cortex v2. Missing pieces: confidence scoring, forgetting curves, nightly consolidation.

Nemotron on Apple Silicon

Nemotron 3 Nano 4B inference works on Apple Silicon (~60+ tok/s, 2.84GB RAM). But all Nemotron fine-tuning requires NVIDIA GPUs (NeMo, CUDA, Slurm). Pre-trained Nemotron is a viable inference option but cannot be domain-adapted on Mac Mini.

4. Persistent Inference Patterns

What It Means for Baker

When the AO PM processes a WhatsApp message about Hagenauer on day 47, it should know everything from days 1-46 without re-reading every transcript. The state is the product.

Three Validated Production Patterns

Pattern 1: Append-Only Event Log (Anthropic Managed Agents)

Sessions are durable, append-only logs external to the context window. getEvents() enables flexible context interrogation. On crash, a new harness boots and resumes.

→ Baker already has this. cortex_events is the append-only log. classify_intent() + capability_runner is the stateless harness.

Pattern 2: Three-Primitive Context Stack (Anthropic API-native)

  1. Tool-Result Clearing (~50K tokens) — replace old tool outputs with [cleared]
  2. Compaction (~150K tokens) — LLM-summarize history
  3. Memory Tool (cross-session) — persistent file storage

→ Baker should adopt for Tier 1. Capability runner calls can use compaction + clearing. Memory tool maps to wiki_pages.

Pattern 3: Checkpoint-Before-Exhaustion

Agents save state to external storage when approaching context limits.

→ Tier 2 (Mac Mini, 200K) should checkpoint to PostgreSQL at ~150K.

7. Open Questions for Director

Decisions Required

  1. Base model: Qwen 3.5 4B vs Gemma 4 E4B?
    Qwen: smaller (3GB), faster, better multilingual. Gemma: already in stack, MoE.
    Recommendation: Start with Qwen 3.5 4B (smaller = faster iteration).
  2. Classification taxonomy?
    Intent labels, priority levels, matter detection scope.
    Needs 1-hour interview session before data prep.
  3. When to build?
    Now (improves Tier 1 immediately) vs after Tier 2 (more training data).
    Recommendation: Build extraction pipeline now, POC on 1K examples. Full model after 2 weeks of Tier 2 data.

Information Needed

  1. Mac Mini RAM: Confirmed 24GB. Upgrade possible? (M4 Pro supports up to 48GB)
  2. Ollama co-existence: Two models (classifier 4B + Gemma 4 12B). ~2-3s cold-start when switching. Acceptable?
  3. Cloud fine-tuning budget: If 4B proves too small → 7B still on Mac Mini. 27B+ needs cloud GPU (~$2-5/hr). Likely not needed.
