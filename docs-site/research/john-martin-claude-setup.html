Claude for Stock Research

A practical setup so research compounds — without bloating the context window
Brisen Capital SA · Geneva 8 May 2026
Bottom line. Claude is excellent for digesting filings, building investment theses, and stress-testing your reasoning — but only if you keep chats short and knowledge in files. Treat conversations as throwaway; treat files as memory. Follow the four-file structure below and you will avoid the single biggest mistake retail investors make with Claude: 50-turn threads where the model forgets, contradicts itself, and bills you for the privilege.
The principle — Chat is the workbench. Project Knowledge is the memory. Files are the durable artifacts.

1. One-time setup (15 minutes)

  1. Subscribe to Claude.ai Pro (USD ~20/month) or Max if you want larger context. Pro is sufficient for individual stock research. Sign up at claude.ai.
  2. Turn on Memory. Settings → Profile → Memory → ON. This lets Claude remember your investor profile across sessions. Useful supplement, not a substitute for the file discipline below.
  3. Set a global system prompt in Settings → Custom Instructions. Suggested text: 
    I am a private investor based in Switzerland. I focus on
[dividend stocks / European equities / your niche]. Risk tolerance:
[moderate / conservative]. When I ask about a company, always
structure the answer as:
  1. Bottom line (buy / hold / avoid — with reason)
  2. Thesis in 3 bullets
  3. Key numbers (revenue, margin, leverage, valuation)
  4. Risks
  5. What would change my view

Never invent figures. If you cannot verify a number from a
primary source, say so explicitly. Do not give legal,
tax or regulatory advice.
  4. Bookmark Perplexity.ai as a secondary tool for time-sensitive news with citations. Use it alongside Claude, not instead.

2. Per-ticker setup (5 minutes per company)

For every stock you actively follow, create a dedicated Project inside Claude. One ticker = one Project. Do not mix companies in a single Project — the knowledge bleeds.

  1. Click + New Project → name it after the ticker (e.g. NESN.SW — Nestlé).
  2. Add Custom Instructions at the Project level (3-5 lines): your specific thesis on this name, your time horizon, your position size if any.
  3. Upload four base files to Project Knowledge (templates in Section 6):
    • thesis.md — your bull case, bear case, base case in three short paragraphs.
    • key-numbers.md — latest reported revenue, EBITDA, net debt, EPS, dividend, guidance.
    • watch-triggers.md — the specific events that would change your view (margin compression below X%, debt above Y, CEO change, regulatory ruling).
    • decision-log.md — every buy/sell/hold decision with date, price, reason. Append-only.
  4. Optionally upload the latest 10-K / annual report PDF and the most recent earnings transcript. Replace quarterly — do not stockpile every filing.

3. Research workflow (per session)

Step Action
Open Project Click into the relevant ticker Project. Confirm Project Knowledge is loaded (Claude can see your four files).
Start a NEW chat Always. Do not continue old threads beyond a single research question. New question = new chat. The Project Knowledge carries over; the chat noise does not.
State the question One concrete question per chat. Examples: "What did Nestlé guide on H2 margin?" "Compare Roche vs Novartis on pipeline replacement risk." "Stress-test my thesis: what if China dairy demand drops 20%?"
Iterate 3-7 turns Refine, push back, ask for sources. Stop when you have your answer. Do not let a chat run past ~10 turns — quality degrades, context bloats.
End-of-chat ritual Ask: "Summarise today's findings as a one-paragraph entry I can append to my decision-log." Copy the output. Edit your decision-log.md locally. Re-upload to Project Knowledge.
Close the chat Walk away. Tomorrow's research starts a fresh chat against the (now updated) Project Knowledge.

4. Hard rules

  1. New chat for every new question. Threads bloat after ~10 turns — Claude starts contradicting itself and you are paying tokens for amnesia.
  2. Live prices come from your broker, never from Claude. Claude analyses; broker quotes. Treat any price or current valuation Claude states as suspect until verified.
  3. Numbers from primary source only. Earnings figures from the company's own filing or transcript — not Claude's recall. Upload the PDF, then ask.
  4. Web Search ON for time-sensitive questions. Off by default in some plans — check the toggle below the message box.
  5. Quarterly clean-up. Once a quarter, review what Memory has stored about you (Settings → Memory → View). Delete anything stale or wrong. Refresh the four files per Project.
  6. Confidentiality. Do not paste private information you would not put on a postcard. Anthropic does not train on Pro/Max chats by default, but treat every prompt as potentially loggable.

5. The four file templates

thesis.md

# [TICKER] — Investment Thesis
Last updated: YYYY-MM-DD
Holding: [yes / no / size]

## Base case
[2-3 sentences. What you expect to happen and why.]

## Bull case
[What goes right. What multiple / earnings / catalyst.]

## Bear case
[What breaks the thesis. Be specific.]

## Time horizon
[6 months / 2 years / 5+ years]

key-numbers.md

# [TICKER] — Key Numbers
Source: [Q1 2026 report, dated YYYY-MM-DD]

Revenue (LTM):       [CHF X bn]
EBITDA margin:       [X%]
Net debt / EBITDA:   [X.Xx]
EPS (LTM):           [CHF X.XX]
Dividend yield:      [X.X%]
Forward P/E:         [Xx]
Guidance (next FY):  [revenue X-Y%, margin Z%]

## My valuation
[Your fair-value estimate and how you got there.]

watch-triggers.md

# [TICKER] — Watch Triggers
What would change my view:

BEAR triggers (sell / trim):
- EBITDA margin drops below [X%] for two consecutive quarters
- Net debt / EBITDA rises above [X.Xx]
- CEO or CFO change
- [specific regulatory or competitive event]

BULL triggers (add):
- Guidance raise on margin
- [specific catalyst]
- Multiple compression to forward P/E below [X]

NEUTRAL events (note, don't act):
- [routine items you don't want to over-react to]

decision-log.md

# [TICKER] — Decision Log

## YYYY-MM-DD — Initial position
Action: BUY 100 shares @ CHF X
Reason: [one paragraph summarising the thesis at entry]

## YYYY-MM-DD — [event]
Action: HOLD / ADD / TRIM / SELL
Reason: [what happened, what you concluded, why]
Source: [link to filing / transcript / news]

[Append-only. Never edit historical entries — you want
the audit trail of how your thinking evolved.]

6. Common pitfalls (and how to avoid them)

Pitfall 1 — The mega-chat

Running one chat for weeks. Symptoms: Claude forgets, contradicts itself, repeats. Fix: new chat per question, always.

Pitfall 2 — Trusting Claude's numbers

Claude confidently invents financial figures (hallucination). Fix: every number must come from an uploaded primary source. Ask Claude to cite the page.

Pitfall 3 — Stale Project Knowledge

Files uploaded six months ago are now wrong. Fix: replace the four files quarterly. Old earnings PDF out, new one in.

Pitfall 4 — Confirmation bias

Asking only for the bull case. Fix: explicitly prompt for the bear case in every research session. The principle: if you cannot articulate the strongest counter-argument, you do not understand the trade.

Pitfall 5 — Memory pollution

Claude's Memory feature retaining a bad assumption. Fix: monthly review of stored memories. Delete what is wrong.

Pitfall 6 — Mistaking Claude for alpha

Claude works on public information. Real edge requires Bloomberg / FactSet / a research broker. Frame Claude as a productivity tool, not an information advantage.

7. A reasonable starting portfolio of Projects

Begin with no more than five Projects. Each takes 30-60 minutes a quarter to maintain properly. Beyond five, the discipline collapses. A practical first set:

  1. Two or three core holdings you already own.
  2. One or two watch-list names you are considering.
  3. One thematic Project (e.g. European Defence, Swiss Dividend Aristocrats) for cross-cutting research.
If in doubt: start with one Project on one stock you know well. Live with the four-file workflow for two weeks. Then expand. The discipline is the moat — not the tool.
