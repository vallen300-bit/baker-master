Current architecture summary

Baker is no longer "a Sentinel + a thinking session + the Director." It is a multi-agent institution in three layers. The Sentinel layer (Baker the always-on service) survives from v5.1 and remains the memory bridge. A new reasoning + orchestration layer — a fleet of Claude Code agents plus a cross-vendor verifier — coordinates over the Brisen Lab bus. The authority layer is unchanged at the top: the Director is final authority, but now talks to AH1, who orchestrates the rest under a strict tiered bank model. Target reasoning model is Cortex 3T (per-matter 6-phase cycles), migration in flight.

Stack deltas vs v5.1: LLM is now Claude Opus 4.8 (claude-opus-4-8), not 4.6. Baker MCP exposes ~47 tools, not 18. FastAPI 8080, Neon Postgres, Qdrant/Voyage 1024d, and Render auto-deploy all hold.

Authority split — the bank model

Tier A

AH1 standing authorization

Acts on gate conditions; logs and reports after.

  • PR merges on green CI + review
  • Mailbox dispatch to B-codes
  • Brief drafting, memory R/W
  • Small infra / docs commits

Tier B

Per-action authorization

Prepare exact action, ask "Shall I X?", wait, execute, log the quote.

  • Render env-var flips
  • External posts (Slack / Gmail)
  • launchd / secret-file PUTs
  • Decisions-log entries

Tier C

Director only

AH1 surfaces the option; never executes.

  • Capital, counterparty drafts
  • Externally-facing speech
  • Third-party logins
  • Acts under Director's name

B-codes, Codex, Architect, and AID carry no Director-facing authority — they feed AH1, who carries the decision to the Director.

Harness V2 implications

1 · The bus is the load-bearing primitive

Not the MCP. V2 must model the Brisen Lab bus as the spine — dispatch, gates, heartbeats, acks all ride it. Any V2 that omits the bus repeats v5.1's central error.

2 · Roles are tiered, not flat

Encode A/B/C authority and the Director-facing-or-not boundary as first-class. A B-code or Codex must be structurally unable to address the Director.

3 · Review is multi-vendor, multi-gate

Treat G0–G3 as the default path. Codex (cross-vendor) + security-review + deputy is the institution's correctness immune system.

4 · Memory is two-layer

Keep the split: Baker MCP (live operational) vs vault curated (per-matter, persistent). Conflating them is how state rots.

5 · Cortex 3T is the reasoning target

V2's reasoning model should assume per-matter cycles (sense → load → reason → propose → act → archive), not one global thinking session.

6 · Cheap build layer, singular orchestration

Let B-codes / Researcher fan out freely; keep orchestration (AH1) singular and stateful.

Recommendation — replace vs new page

Replace baker-v5 with this v6 page; keep the old URL as a "superseded" banner + redirect.

v5.1 is not merely incomplete — it is actively misleading (three actors, 18 tools, no bus). A new page that leaves v5.1 live as a peer lets the wrong mental model keep circulating. Keeping the old URL alive as a redirect/banner preserves links and version history. Director ratified: replace as v6 (2026-05-31).

Counter-case (devil's advocate): if the Harness V2 actor set may shift again in weeks, freezing v6 as final risks a third rewrite. Mitigation: v6 is marked "current as of 2026-05-31, Harness V2 baseline — subject to revision" rather than final. A stale-but-honest page costs far less than a confidently-wrong one.
