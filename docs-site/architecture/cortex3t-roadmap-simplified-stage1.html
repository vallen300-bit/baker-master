How we got here — 5 layers of research

L1
Are other companies using Anthropic Memory?
Real adopters (Netflix, Rakuten, Wisedocs, Ando) — but enterprise pilot scope, very recent (April 2026 public beta), partly the Managed Agents version we wouldn't use.
L2
What are the alternatives?
Mem0 / Letta / Zep-Graphiti / Cognee / pgvector DIY. Brisen profile (few entities, deep history, audit-grade) fits Zep-Graphiti or DIY-Postgres best.
L3
What does Brisen already have?
~70% of cortex_state already built: capability_threads + capability_turns + feedback_ledger + signal_queue + ao_project_state + Citations API + autonomy_level routing.
L4
What does the orchestrator code already do?
chain_runner.py implements 6-step pipeline (qualify → plan → execute → verify → log → report) mapping almost 1-to-1 onto Cortex phases. Reframe: Cortex-3T = chain_runner + persistence + named formalization.
L5
What does one cycle on a real AO signal actually do?
Concrete simulation on alert #15725 ("В мае штраф 3 миллиона") — ~50 sec total, ~$0.08, 6 phases, full Citations API trail, Tier B-gated proposal in Director's hand within ~1 minute.
Plus Director's design pivot: refresh-driven by Director, not by scheduler. Removes polling/auto-re-fire scope; replaced with single 🔄 Refresh button on the proposal card. Director-led freshness, production-data-led automation.

The 6-phase Cortex cycle (mapped onto existing Brisen code)

PHASE 1 · SENSE
Pull triggering signal + correlate with related signals (last 30d)
Reuses: signal_queue, existing pipeline.py
PHASE 2 · LOAD
Load matter state + wiki + threads + recent activity
Reuses: capability_runner.py 25+ loaders. NEW: 3 SQL queries pulling Director outbound + named-entity inbound + baker_actions (last 14d) — situational awareness extension.
PHASE 3 · REASON
Claude Opus call with cached context, hypothesis ranking, draft response
Reuses: chain_runner._generate_plan, prompt cache (M0 PROMPT_CACHE_AUDIT_1). NEW prompt clause: "Check recent activity before proposing duplicates."
PHASE 4 · PROPOSE
Generate Tier-B proposal package: hypothesis + draft + actions + citations
Reuses: Citations API (M0 PR #59). NEW: 4-button proposal card model.
PHASE 5 · ACT
Wait for Director Gold; on Approve, send response + execute internal actions
Reuses: chain_runner._execute_plan, _notify_director, baker_actions audit. NEW: final freshness check on Approve.
PHASE 6 · ARCHIVE
Persist cycle, write feedback_ledger, update matter state
Reuses: feedback_ledger (CHANDA §2 Leg 2), GOLD workflow (PR #66), ao_project_state, capability_threads.

The two new Postgres tables

Additive, idempotent, follow existing migration patterns. UUID primary keys. JSONB payloads. CASCADE delete on parent reference. ~2 hours of migration work.

-- cortex_cycles: one row per Cortex reasoning cycle CREATE TABLE IF NOT EXISTS cortex_cycles ( cycle_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), matter_slug TEXT NOT NULL, triggered_by TEXT NOT NULL, -- 'signal' / 'director' / 'refresh' / ... trigger_signal_id BIGINT, started_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), completed_at TIMESTAMPTZ, last_loaded_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), current_phase TEXT NOT NULL DEFAULT 'sense', status TEXT NOT NULL DEFAULT 'in_flight', proposal_id UUID, director_action TEXT, -- 'gold_approved' / ... feedback_ledger_id BIGINT, cost_tokens INTEGER DEFAULT 0, cost_dollars NUMERIC(10,4) DEFAULT 0 ); -- cortex_phase_outputs: per-phase artifacts within a cycle CREATE TABLE IF NOT EXISTS cortex_phase_outputs ( output_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), cycle_id UUID NOT NULL REFERENCES cortex_cycles(cycle_id) ON DELETE CASCADE, phase TEXT NOT NULL, -- 'sense' / 'load' / 'reason' / ... phase_order INT NOT NULL, artifact_type TEXT NOT NULL, payload JSONB NOT NULL, citations JSONB DEFAULT '[]'::jsonb, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW() );

What is explicitly NOT in Stage 1

The following items remain open for Stage 2 and were NOT ratified in this session. Stage 2 design happens after Stage 1 ships and produces real production data.

  • Other matters beyond AO (MOVIE AM / NVIDIA / Corinthia / Hagenauer ingestion) — open question
  • Model B integration (Cortex-3T running above existing Pattern-2 capabilities like AO PM) — open question
  • Repair + Cortex optimization lanes in M2' — open question
  • Polling / scheduled re-evaluation — Director-driven refresh proven sufficient for V1
  • Auto re-fire on new related signal — replaced by manual refresh button
  • Eval suite, model bumps, retention, trust audit — production data tells us what's needed
  • Game Theory injection / Counterparty Game Advisor — AO+MOVIE manual injection already running
  • Graphiti adoption (temporal knowledge graph layer) — evaluated later if production needs surface
  • Anthropic Memory tool integration — Brisen has equivalent built; no value-add
  • Dashboard UI for proposal cards — Slack rendering for V1; same model surfaces in Dashboard later with zero rewrite
