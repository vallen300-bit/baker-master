§5Counter-case — devil's advocate

Auto-fired on every roadmap. 25+ briefs is a lot of forward planning before first cycle ships. Three failure modes:
1.Roadmap-as-ceremony
M0's success doesn't predict M2-M6. Those involve genuinely novel reasoning-loop integration. Risk: weeks on M1/M2 polish before learning M3 is wrong.
Mitigation: keep M2 minimal. Prefer M3 dispatch on stub matter early to surface integration issues before scaffolding hardens.
2.Wiki + Cortex stream divergence
Two parallel streams can drift. M1 ships wiki without knowing what cortex actually needs to read; M2 ships cortex assuming wiki shape that isn't there.
Mitigation: every M1/M2 brief must cite which M3 capability it enables. If unclear, defer the brief.
3.Hagenauer pick
First real Cortex-3T cycle on a live €600K dispute is high-stakes for a brand-new system. If Cortex misadvises, real money moves wrong.
Mitigation: M3 first cycles MUST be Tier B (Director-gated). No Tier A on Hagenauer until M5 gating closes.
