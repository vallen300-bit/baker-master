What it does

In one sentence You name an issue or goal; the agent builds the context, analyses it with Sacca, and produces 1–3 page whiteboard documents for your human discussion group to verify, amend and correct — then carries the ratified step plan forward with a daily progress email.

Your 4 touchpoints — everything else runs without you

1 · Lock

  • Ratify the lock card
  • Issue, group, owner, channels

2 · Triaga V1

  • Ratify the plan
  • Pick pager size 1/2/3 + output types

3 · Triaga V2

  • Ratify the revised plan
  • After research + comments

4 · Final

  • Ratify Output V2 + step plan
  • Plan goes live only then
WHAT THE GROUP GETS
1–3 page whiteboard pagers (HTML + PDF) to discuss, plus a Word copy with track changes — their written amendments flow back into the next version automatically.

What happens inside the agent-only phases

PhaseWhat the agent does, alone
1 · ContextSweeps EVERY Brisen source — email, WhatsApp, meeting transcripts, vault, ClaimsMax, news feeds, tasks, calendar — logs what each returned, files it all into the project room, names the gaps.
2 · SaccaRuns both frames: problem (what / origin / good state / path) and opportunity (today / resources / tomorrow / path). Every line labeled known, assumed, or needs-research.
4 · ResearchParallel research on the needs-research list; ingests the group's track-change comments; synthesizes everything into one memo — conflicts decided, never averaged.
6 · AftercareOne daily rollup email across ALL active step plans: done / due today / overdue / blocked. Whoever opens the matter next also reports drift. Room never freezes.

Built-in safety rules

No skipped gates

  • No output before its Triaga
  • Plan live only after final OK

No silent agents

  • New desk head only on your order
  • No desk = open question to you

No auto-send outside

  • Externals: you always press Send
  • Internal auto-send only if you authorize
TO START A RUN
