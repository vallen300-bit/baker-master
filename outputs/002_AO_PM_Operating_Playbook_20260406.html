What AO PM Does
Single source of truth for everything AO
AO PM is Baker's dedicated intelligence officer for Andrey Oskolkov. It maintains persistent state, reads 6 compiled view files at every invocation, monitors signals across WhatsApp/email/meetings, and now learns from every interaction via the knowledge compounding loop.

It challenges assumptions, flags risks, presents counter-arguments. Not a yes-man.
Monitors
Communication gap, AO mood, capital call deadlines, relationship health, Hagenauer cross-impact
Signals
WhatsApp (AO + orbit), email (Buchwalder, Constantinos), meetings, documents
Learns
Extracts insights after every Q&A, stages for your review, compounds knowledge weekly
Day-to-Day: Asking AO PM Questions
How to invoke
Just ask Baker anything about AO. The classifier routes AO-related queries to AO PM automatically.

CEO Cockpit: Type in the chat bar. Slack: Message Baker directly.
Trigger words: AO, Oskolkov, Andrey, capital call, Aelio, Aukera, Buchwalder, Constantinos
Example queries:
• "What's the current state of the capital call?"
• "When did we last hear from Buchwalder?"
• "Prepare me for a call with AO about the restructuring"
• "What are the risks if AO doesn't fund by May?"

What happens under the hood:
1. Baker loads all 6 view files (psychology, channels, rules, sensitive issues, communication rules, agenda)
2. Injects live state from PostgreSQL (last contact, open actions, red flags)
3. Shows pending insights so PM doesn't re-discover known facts
4. Opus reasons over the full context and responds
5. After response: auto-extracts state updates + wiki-worthy insights (fire-and-forget)
Debrief Sessions — Path 1: Director Writes Directly
Debrief Flow
Director has new info → Tells Baker via Cowork → Baker writes to view file → Git commit → PM reads updated view on next invocation
When to debrief
After any significant AO interaction: a call, a meeting, a key WhatsApp exchange, a decision, a mood shift. Don't wait for a scheduled session — debrief while it's fresh.
1
Open Cowork (Claude Desktop)
Cowork has access to Baker's MCP tools and can read/write the view files. Use Cowork for debriefs, not CEO Cockpit — debriefs need full file editing capability.
2
Tell Baker what's new
Be direct. Examples:

"Update psychology.md — AO mentioned he's worried about the Aukera timeline. Add to drivers: time pressure on restructuring is creating anxiety."

"Update agenda.md — Capital call is now parked until after the Aukera facility closes. Move it to parked matters."

"Update communication_rules.md — AO responds better to 'ammunition' framing than 'obligation' framing for capital calls. Confirmed in WhatsApp today."

Baker writes the edit directly into the .md file and commits. No staging, no pending queue. Your word is gospel.
3
Verify the update
Ask Baker to show you the updated section: "Read back the Framing section of communication_rules.md."
If it's wrong, correct it immediately.
What goes in which file
psychology.md — How AO thinks, what drives him, fears, motivations, personality patterns
investment_channels.md — Financial positions, entity structures, amounts, terms
sensitive_issues.md — Minefields: what NOT to say, what to avoid, red lines
communication_rules.md — How to talk to AO: framing, timing, tone, Rule Zero
agenda.md — What's active, what's parked, next actions

Not sure? Ask Baker: "Where should I file this?"
Knowledge Compounding — Path 2: Baker Discovers, You Approve
Self-Learning Loop
You ask AO PM a question → Opus reasons + responds → Opus extracts wiki insights → pm_pending_insights (DB) → You review → View files get smarter
1
Insights appear automatically
Every time you ask AO PM a substantive question, Baker extracts wiki-worthy insights in the background. You don't need to do anything — it happens automatically.

Daily briefing will tell you: "KNOWLEDGE COMPOUNDING: 3 pending insight(s) awaiting your review."
2
Review pending insights
Ask Baker (via CEO Cockpit, Slack, or Cowork):

"Show me pending insights" or "What insights are waiting?"

Baker will list them with IDs, target files, dates, and confidence levels:

#14 [2026-04-06] AO responds better to "ammunition" framing than "obligation" framing for capital calls → Target: communication_rules.md (confidence: high)
#15 [2026-04-06] Buchwalder sends updates on 1st of each month → Target: agenda.md (confidence: medium)
3
Promote or reject
To promote: "Promote #14"
Baker responds: "Insight #14 approved. Queued for Code Brisen to merge into communication_rules.md. It will appear in your next view file update."

To reject: "Reject #15 — that's not a reliable pattern yet"
Baker stores your reason. The extraction model learns NOT to re-discover this insight.

To skip: Just leave it pending. No action needed. It stays in the queue.
Important: Promote does NOT instantly edit view files
"Promote" marks the insight as approved in the database. The actual view file edit happens when Code Brisen (or AI Head) picks up approved items and commits them. This is by design — view files only change through git commits, ensuring traceability and rollback ability.

If you want an insight in the view file RIGHT NOW, use the debrief path instead: "Update communication_rules.md — add this: [insight text]"
Daily Briefing — What You See
The daily briefing's AO section checks:

1. Communication gap — Days since last outbound to AO. Warning at 10 days. Critical at 14 days (Rule Zero).
2. Pending discussion items — Count of topics queued for next AO interaction.
3. Approaching deadlines — AO-related deadlines in next 14 days (capital calls, Aukera facility, etc.).
4. Pending insights — Count of knowledge compounding items awaiting your review.
5. Cross-matter alerts — Hagenauer/RG7 developments that affect AO.

Only appears in briefing if there's something actionable. No noise.
