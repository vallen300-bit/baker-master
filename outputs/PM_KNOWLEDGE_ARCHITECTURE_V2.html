Two paths into the wiki
Path 1: Director Debrief
Director tells Baker something new → Baker writes it directly into the view file → git commit.

Fast. Authoritative. No staging needed.

"Baker, the GM changed. Update operator_dynamics.md — new GM is Thomas Berger, started March 2026."
Path 2: Discovered Knowledge
Baker discovers something during a Q&A or signal detection → writes to _pending.md → Director reviews → promote or reject.

Safe. No autonomous wiki corruption.

"Found: Mario Habicher sends P&L by 3rd of month (observed 3 consecutive months). Promote to kpi_framework.md?"
Self-Healing
Periodic Lint Pass
Baker audits the wiki against raw sources and recent signals:

• Contradictions — view file says X, recent email says Y
• Stale data — figure from 6 months ago, newer data exists
• Gaps — topic mentioned but no page covers it
• Unverified — claim without source citation

Results → _pending.md for Director review

Four Rules
1
Wiki is gospel
If anything contradicts the view files, the view files win. Everything else is input or scratchpad.
2
Director writes directly
Debrief sessions go straight into view files. No staging needed. Baker commits immediately.
3
Baker proposes, Director approves
Discovered knowledge goes to _pending.md. Baker never writes to view files autonomously. Director promotes or rejects.
4
The wiki heals itself
Periodic lint pass finds contradictions, stale claims, gaps. Results go to _pending.md for review.
