# Brief: D7 — Morning Brief v2 (Interactive Action Cards)

**For:** Code Brisen (frontend)
**From:** AI Head (Session 27)
**Priority:** High
**Effort:** L (1-2 days)
**Depends on:** Nothing — backend proposals already exist

---

## Problem

The morning brief is text-only. Baker generates proposals ("Dismiss these 5 stale alerts", "Prepare for your meeting with Ofenheimer") but the Director has to manually go to the right tab and do each action. The brief should be **interactive** — click a button, Baker executes.

---

## Current State

The `/api/dashboard/morning-brief` endpoint already returns:

```json
{
  "narrative": "3 fires overnight, Hagenauer deadline approaching...",
  "proposals": [
    {

    },
    {
```

The `proposals` array is generated by Haiku but **completely ignored by the frontend**. The narrative is rendered as plain text.

## What to Build

### 1. Action Cards (below the narrative)

Each proposal becomes a clickable card:

```
┌──────────────────────────────────────────────┐
│ 📋 Prepare for Ofenheimer meeting      [Go →]│
│ Baker will search memory and draft a brief    │
└──────────────────────────────────────────────┘
┌──────────────────────────────────────────────┐
│ 🧹 Dismiss 5 stale T3 alerts        [Do it →]│
│ These alerts are >48h old with no action      │
└──────────────────────────────────────────────┘
┌──────────────────────────────────────────────┐
│ 📧 Draft follow-up to Sandra          [Go →] │
│ Last contact 12 days ago (4x normal cadence)  │
└──────────────────────────────────────────────┘
```

### 2. Action Types

| Action | Button Label | What happens on click |
|--------|-------------|----------------------|
| `analyze` | "Go →" | Opens chat tab, pre-fills the question, auto-sends |
| `dismiss` | "Do it →" | Calls `/api/alerts/bulk-dismiss` inline, shows checkmark |
| `draft` | "Draft →" | Opens chat tab, sends "Draft an email to X about Y" |
| `specialist` | "Ask →" | Opens specialist tab, selects capability, pre-fills question |
| `call` | "Details →" | Expands card to show contact details + phone number |

### 3. Visual Design

- Cards sit in a horizontal scrollable row below the narrative (mobile: vertical stack)
- Each card: white background, subtle left border color by action type (blue=analyze, green=dismiss, orange=draft)
- Compact: 2 lines max. Label (bold) + sublabel (gray, 13px)
- Button: pill shape, right-aligned, matches action color
- After execution: card fades to green checkmark + "Done"
- Max 5 cards shown (Haiku already limits proposals)

### 4. Click Handlers

**`analyze` / `draft`:**
```js
// Switch to chat tab, inject question, auto-send
switchTab('chat');
var input = document.getElementById('chatInput');
input.value = proposal.params.question;
sendChat(); // triggers the scan
```

**`dismiss`:**
```js
// Inline API call, no tab switch
await bakerFetch('/api/alerts/bulk-dismiss', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ tier: proposal.params.tier })
});
card.classList.add('done');
card.querySelector('.action-btn').textContent = '✓ Done';
refreshAlertBadge(); // update fire count
```

**`specialist`:**
```js
switchTab('specialist');
var picker = document.getElementById('capPicker');
picker.value = proposal.params.capability_slug;
picker.dispatchEvent(new Event('change'));
var input = document.getElementById('specialistInput');
input.value = proposal.params.question;
sendSpecialist();
```

### 5. Mobile

Same cards but vertical stack (full width). The mobile morning brief doesn't exist yet — this is desktop only for now. Mobile gets it in a future brief.

## Acceptance Criteria

1. Morning brief shows action cards below narrative
2. "Go →" opens chat and auto-sends the question
3. "Do it →" executes inline (dismiss, etc.) with visual feedback
4. Cards show correct label + sublabel from Haiku proposals
5. After execution, card shows "Done" state
6. Max 5 cards, horizontally scrollable on desktop
7. Works with existing `/api/dashboard/morning-brief` response — no backend changes

- Proposal editing (Director can't modify the question before sending — just clicks "Go")
- Proposal persistence (cards regenerate on page refresh)
