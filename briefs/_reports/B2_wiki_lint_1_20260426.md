## What shipped

Single-entrypoint weekly wiki health check: `kbl/wiki_lint.py` + 7-check
package under `kbl/lint_checks/`. APScheduler job
`_wiki_lint_weekly_job` registered in `triggers/embedded_scheduler.py`
behind `WIKI_LINT_ENABLED` env flag (default **false** — code ships
dormant; Director flips after first dry-run).

| Check | Severity | Type |
|---|---|---|
| 1. retired_slug_reference | error | deterministic |
| 2. missing_required_files | error / warn (grandfathered) | deterministic |
| 3. orphan_matter_dir | warn | hybrid: filesystem + `signal_queue` |
| 4. one_way_cross_ref | warn | deterministic |
| 5. stale_active_matter | warn | LLM (Gemini 2.5 Pro) |
| 6. contradiction_within_matter | warn | LLM (Gemini 2.5 Pro) |
| 7. inbox_overdue | info | deterministic |

LLM helper: uses `orchestrator.gemini_client.call_pro` directly
(production-wired, zero integration cost, Director Q1 ratified
2026-04-26 overriding RA's Haiku-4.5 recommendation). Cost logging via
`orchestrator.cost_monitor.log_api_cost`. Tests inject stubs for
deterministic LLM behavior.

Hard ceiling: total Gemini Pro tokens per run capped via
`WIKI_LINT_TOKEN_CEILING` (default 200K). On overrun the runner sets
`_aborted = "token_ceiling_exceeded"`, skips remaining LLM checks, and
the report carries a banner.
