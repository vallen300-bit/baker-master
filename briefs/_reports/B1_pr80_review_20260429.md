## ¬ßK ‚Äî Non-blocking observations

1. **Endpoint uses `HTMLResponse` + `status_code` instead of `HTTPException` raise.** The brief's checklist item C-7 ("HTTPException re-raise present (no wrap-into-500 bug)") expects PR #78's idiom. PR #80 uses a different but equally safe idiom: every helper (`verify_token`, `already_decided`, `record_decision`, `lookup_matter_slug`) self-contains errors (returns sentinel on failure rather than raising), and the endpoint returns explicit `HTMLResponse(..., status_code=403/404/503/200)`. There is no outer try/except wrapper, so there is no wrap-into-500 risk to defend against. **Note-only ‚Äî equally safe, just different.**

2. **`record_decision` audit insert is best-effort (logs + swallows on failure).** If a Director's approve-tap audit insert fails (e.g. transient PG error), the BackgroundTask still fires the cycle and the cycle's `cortex_cycles` row provides the durable record ‚Äî but no `baker_actions` row will exist. Trade-off accepted in module docstring (line 126-127): "audit best-effort". The cycle audit trail comes from `cortex_cycles` itself, not `baker_actions`. **Note-only.**

3. **Slack-outage policy is "no runaway spend".** When `post_gate` returns False due to a transient Slack post failure (with secret SET and not already-decided), `maybe_trigger_cortex` emits a `logger.warning` and **skips the cycle** rather than falling through to direct-fire. Deliberate by design (no auto-spend without Director approval), but worth flagging to A: a Slack outage will silently suppress Cortex cycles in the gate-enabled regime. Operator can use `CORTEX_GATE_ENABLED=false` as kill-switch if Slack is down for an extended period. **Note-only.**

4. **Schema deviation acknowledged in module docstring.** Brief used `signal_text` / `matter_slug` column names; actual `signal_queue` schema uses `summary` / `matter` (per `memory/store_back.py:6600`). B2 corrected this in the gate module's docstring (lines 15-20) and used the real names throughout. Lesson #40 cousin (verify schema before referencing) ‚Äî handled correctly. **Already documented; note-only confirmation.**
