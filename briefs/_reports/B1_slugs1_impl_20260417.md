## Public API (matches task §2 + aliases_for added per §3c)

```python
registry_version() -> int
canonical_slugs() -> set[str]           # all regardless of status
active_slugs()    -> set[str]           # status == active only
is_canonical(slug: str | None) -> bool  # None always True
normalize(raw: str | None) -> str | None
describe(slug: str) -> str              # raises KeyError if unknown
aliases_for(slug: str) -> list[str]     # raises KeyError if unknown
reload() -> None
```

`SlugRegistryError` raised on: missing `BAKER_VAULT_PATH`, missing `slugs.yml`, malformed YAML, wrong top-level shape, unknown status, duplicate canonical slug, duplicate alias across slugs, alias colliding with any canonical slug.

Cache is module-level, protected by `threading.Lock`. `reload()` drops it.

## Semantic changes (documented)

### `run_kbl_eval.py` — prompt

- **Before:** prompt hardcoded 6 example slugs (`hagenauer-rg7, cupial, mo-vie, ao, brisen-lp, mrci`).
- **After:** `_build_step1_prompt()` enumerates all 19 `active_slugs()` as a bulleted list, embeds Director's verbatim vedana rules (per B3 task §1c), and explicitly permits `null` while rejecting generic category strings.
- B3's first eval report attributed the 70% matter-miss rate to the 6-example enum. This patch addresses the structural cause; B3's D1 retry will measure it.

### `run_kbl_eval.py` — score_row invariant preservation

Old `normalize_matter()` returned `raw.lower().strip()` for unknown inputs (so a garbage output `"hospitality"` stayed as `"hospitality"`, never matching `None`). New `slug_registry.normalize()` returns `None` on unknown — which would spuriously match labels of `None`.

Added explicit guard in `score_row`:

```python
model_returned_non_empty = (
    isinstance(out_matter_raw, str)
    and out_matter_raw.strip() != ""
    and out_matter_raw.strip().lower() not in {"none", "null"}
)
unknown_non_canonical = model_returned_non_empty and out_matter is None
matter_ok = (out_matter == label_pm) and not unknown_non_canonical
```

Net: a model spewing `"hospitality"` on a `primary_matter_expected: null` signal scores **False** (pre-refactor behavior preserved).

### `run_kbl_eval.py` — results JSON

Added `slugs_version` field to the output JSON (design note §4). Comparing runs across registry versions is now honest.

### `build_eval_seed.py` — hint coverage expansion

Per task §3c's literal spec: `{slug: [slug, slug.replace("-"," "), *aliases] for slug in active_slugs()}`. This iterates ALL 19 active slugs, not just the 9 previously in `MATTER_HINTS`. Net: hint coverage expands; the 9 previously hinted slugs keep every prior keyword. Iteration sorted for determinism (affects which slug wins ambiguous-signal hints — cosmetic, never authoritative).
