### N2 — Director-organization strip uses substring match (false-positive risk)

**Location:** [`kbl/steps/step3_extract.py:217-224`](../../kbl/steps/step3_extract.py).

```python
def _is_director_org(entry: dict[str, Any]) -> bool:
    name = str(entry.get("name", "")).lower().strip()
    if not name:
        return False
    for token in _DIRECTOR_COMPANY_TOKENS:
        if token in name:
            return True
    return False
```

With `_DIRECTOR_COMPANY_TOKENS = {"brisen", "brisengroup", "brisen group"}` and substring match, **any** org name containing "brisen" as substring gets stripped. False positives:
- `"unbrisen corp"` — would be stripped (unlikely in practice)
- `"Brisen-Watson"` — would be stripped (possible joint venture / counterparty name)

The `brisengroup` + `brisen group` tokens are redundant — both are already substring-matched by `brisen` alone.

**Fix (optional).** Word-boundary regex:
```python
_DIRECTOR_COMPANY_PATTERNS = [re.compile(r"\b" + re.escape(t) + r"\b", re.IGNORECASE) for t in ("brisen", "brisen capital", "brisen development")]
```

Or keep substring semantics but drop the redundant tokens (`brisengroup`, `brisen group` already covered by `brisen`).

Defer. Phase-1 real signals won't have "unbrisen" entities; revisit only if a false positive is observed.

### N3 — ISO date regex accepts impossible dates

**Location:** [`kbl/steps/step3_extract.py:72`](../../kbl/steps/step3_extract.py).

```python
_ISO_DATE_RE = re.compile(r"^\d{4}-\d{2}-\d{2}$")
```

Accepts `2026-13-45` (invalid month/day). Not actually ISO-8601-validated. Gemma tends not to emit impossible dates, but a tighter check would foreclose the class:

```python
try:
    datetime.strptime(date, "%Y-%m-%d")
except ValueError:
    return None
```

Adds one import and ~3 lines. Replaces the regex check. Same asymptotic cost.

Low severity. Phase 1 will surface it if Gemma emits bad dates; can harden then.

### N4 — Currency whitelist covers 4 major currencies only


Director's portfolio context (Vienna, Austria; Switzerland home; Germany investments; UHNW network touching Hong Kong / Singapore / Dubai) may surface JPY, AUD, CAD, SGD, HKD, AED eventually. Signals with those currencies get their money entry dropped silently.

**Defer-but-track.** Add when observed in shadow-mode burn-in. Current whitelist is tight and conservative — the cost of missing a money entry is low (Director re-reads signal), the cost of hallucinated currency is higher (misleading frontmatter downstream). Conservative is right for v1.

### N5 — `_clean_money` accepts `NaN` / `inf` (edge case)

**Location:** [`kbl/steps/step3_extract.py:258-274`](../../kbl/steps/step3_extract.py).

`isinstance(amount, (int, float))` correctly rejects `bool`, strings, None. But `float("nan")` / `float("inf")` pass the isinstance check. In practice Gemma doesn't emit JSON `NaN` (not valid JSON) but `json.loads("1e1000")` → `inf`.

**Fix (optional).** Add `math.isfinite(amount)` check:
```python
if isinstance(amount, bool) or not isinstance(amount, (int, float)) or not math.isfinite(amount):
    return None
```

Extremely edge. Defer unless Gemma ever emits literal `Infinity` in a money amount.

## 8. Pattern observations

### State-machine constants

```python
_STATE_RUNNING = "extract_running"
_STATE_NEXT = "awaiting_classify"
_STATE_FAILED = "extract_failed"
_RETRY_BUDGET = 1
```

Every state value is a named constant at module level. No magic strings in the flow. ✓ Matches PR #10's pattern. Consistent across step impls.

### `bool`-before-`int` check on money amount

```python
if isinstance(amount, bool):
    return None
if not isinstance(amount, (int, float)):
    return None
```

Correct — Python's `bool` is a subclass of `int`, so `isinstance(True, int)` is `True`. The explicit pre-check is the right guard against Gemma emitting `{"amount": true}`. Subtle correctness detail handled. ✓

### Frozen dataclass with tuple-of-dict storage

```python
@dataclass(frozen=True)
class ExtractedEntities:
    people: tuple[dict[str, Any], ...] = field(default_factory=tuple)
    ...
    def to_dict(self) -> dict[str, list[dict[str, Any]]]:
        return { "people": [dict(p) for p in self.people], ... }
```

Frozen dataclass prevents accidental caller mutation. `to_dict` copies each entry via `dict(p)` for defense-in-depth against shared-reference mutation. Strong hygiene.

### Cumulative-latency cost ledger

```python
total_latency_ms = 0
while True:
    start = time.monotonic()
    envelope = call_ollama(prompt, ...)
    total_latency_ms += int((time.monotonic() - start) * 1000)
    ...
```

Cumulative latency across retries is the right accounting — operator dashboard sees total time-to-result for a signal, not per-attempt timing. ✓
