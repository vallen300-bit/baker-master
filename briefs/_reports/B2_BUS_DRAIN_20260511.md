## Brief-vs-implementation deltas (V0.2 fold script body was stale; folds applied per fold rationale + verification)

The brief's V0.2 fold prose + §Verification step 5 explicitly demanded four
changes, but the script body in §Implementation step 1 was not updated when

unambiguous; the literal-script interpretation produces a hook that KeyErrors
every drain). Deltas:

| V0.2 fold | Brief script body | Implementation |
|-----------|-------------------|----------------|
| B1 (env vars on python3) | `\| python3 -c '...uses os.environ[SLUG]...' \| BAKER_ROLE=... _emit` (vars on _emit pipe-tail; python3 KeyErrors) | `RESP=... SLUG=... DAEMON_URL=... BAKER_ROLE=... SINCE=... python3 -c '...' \| _emit` (vars on python3 directly; RESP plumbed via env-var, not stdin) |
| B2 (atomic state write) | `with open(state_file, "w") as f: f.write(newest)` (non-atomic) | `tempfile.mkstemp + os.replace`; on OSError leave canonical file unchanged and append "state-file atomic write failed" status line |
| B4 (curl --max-time 4) | `curl --max-time 6` + "timeout 6s" error string | `curl --max-time 4` + "timeout 4s" error string (matches §Verification step 4 expected text) |
| Token budget (limit=50 + RENDER_CAP=30) | `limit=200`, no rendering cap | `limit=50` curl query + `RENDER_CAP = 30` in python3 block + overflow line "(rendering first 30; N more elided — widen `since` to see all)" |

Additional implementation note (not in brief; engineering judgment):

- **`f"...{d[\"detail\"]}..."` would be a Python SyntaxError** ("f-string expression part cannot include a backslash"). The brief script uses backslash-escaped double-quotes inside f-string expressions throughout the python3 block — every `{m["id"]}`, `{d["detail"]}`, `{m["acknowledged_at"] or "no"}`, etc. Discovered while debugging an early test failure (`bash -x` showed `SyntaxError`; whole python3 block fails to parse, hook emits no rendered output, ALL drain attempts silently fail with stdout empty). Refactored every offending line to use `.format()` instead. Functionally identical output.

- **Canonical-source-of-truth placement.** Brief §Files NOT touched does not list `tests/fixtures/`; brief §Files Modified lists `~/.claude/hooks/session-start-bus-drain.sh` as "NEW (user-global, not in any repo)". For testability + drift detection, I shipped the canonical script at `tests/fixtures/session-start-bus-drain.sh` (in-repo) and deployed a byte-identical cp at `~/.claude/hooks/session-start-bus-drain.sh`. Test `test_user_global_matches_repo` enforces zero drift; skips when the deployed file is absent (CI safe).
