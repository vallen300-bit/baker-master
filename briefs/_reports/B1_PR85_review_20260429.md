## §1 — Cost-gate bypass surface analysis (RA-24 Focus #1)

**Question:** can any code path reach the Slack post bypassing the new
whitelist?

**Trace:**

- `post_to_channel` (the Slack call) is reached from exactly one site in
  `triggers/cortex_pre_review_gate.py:359`.
- `post_gate` is the only function that calls `post_to_channel` for the
  Cortex gate.
- `post_gate` has exactly one caller: `triggers/cortex_pipeline.py:69`
  (inside `maybe_trigger_cortex`). Verified via grep.
- Inside `post_gate` the whitelist check (`matter_has_cortex_config`) sits
  on lines 316-321, **before** the secret check (323), token issuance
  (332-333), URL build (334-341), preview fetch (343), text format
  (344-350), and the `post_to_channel` call (359-363).

**Verdict:** ✅ No bypass surface for the Slack-post path. Every successful
return-True from `post_gate` traverses the whitelist gate.

### §1a — Secondary observation: whitelist is gate-scoped, not direct-fire-scoped

While the *Slack post* path is fully gated, the *Cortex spend* path is not.
Trace through `triggers/cortex_pipeline.py` lines 64-99 for a no-config
matter when `CORTEX_GATE_SECRET` is unset/short:

1. `maybe_trigger_cortex` enters gate branch (line 64: `_gate_enabled()` → True default).
2. `post_gate(...)` returns False — whitelist failed (path B inside `post_gate`).
3. Caller cannot distinguish path B (whitelist failed) from path C (secret missing) — both return False.
4. Line 78: `if already_decided(signal_id):` → None → skip.
5. Line 81: `if _secret() is None:` → **True** → executes `pass` (line 84).
6. Falls through past the gate block to legacy direct-fire (lines 102-110).
7. `maybe_run_cycle` fires for a matter with no `cortex-config.md` →
   Phase 2 has no per-matter brain to load → garbage cycle, real spend.

**Pre-existing behavior.** Brief documents this intentional fall-through:
"caller falls through (legacy direct-fire still respects `CORTEX_LIVE_PIPELINE`
so this stays safe)." But the brief's *"single source of truth: presence of
cortex-config.md"* framing is true only **inside the gate**. The whitelist
does NOT cover the misconfigured-secret legacy fallback.

**Severity:** MEDIUM observation, NOT a blocker for this PR. Already-deployed
behavior; PR scope is gate hardening, not direct-fire hardening. Worth
tracking for a future Wave 1 follow-up: either (a) move the whitelist
check into `maybe_trigger_cortex` upstream of the gate-vs-direct-fire fork,
or (b) document explicitly that misconfigured-secret bypasses the whitelist
in production runbooks.

## §2 — Path-traversal on `matter_slug` (RA-24 Focus #2)

**AI Head A flagged Low; B1 confirms Low.**

`matter_has_cortex_config` and `_read_cost_estimate` build:

```python
cfg = root / "wiki" / "matters" / matter_slug / "cortex-config.md"
```

Tested attack: `matter_slug = "../../etc/passwd"`. Path resolves to
`<vault>/etc/passwd/cortex-config.md`. Two real-world constraints:

1. **`matter_slug` is internal-only.** Source = `signal_queue.matter` column
   populated by `kbl/bridge/alerts_to_signal.py`'s matter-routing logic —
   NOT external user input. A signal author cannot dictate the slug.
2. **Operations on the path are bounded:** `is_file()` is a boolean check
   only; `read_text()` happens only inside `_read_cost_estimate` after
   `is_file()` succeeds. Even if traversal landed on a real file, the only
   data extracted is one line beginning with `cost_estimate_dollars:` →
   `float()` parse → returns the float (or default on ValueError). No
   content is logged, returned, or echoed to Slack beyond the float.

**Worst case:** an attacker who could plant a `cortex-config.md` *and*
inject a malicious matter_slug *and* control the float in the planted file
could shift the displayed Slack cost ($X.XX). Not a security issue.

**Verdict:** ✅ Low confirmed. No mitigation needed for this PR.

## §3 — Race condition: `matter_has_cortex_config` ↔ `post_gate` (RA-24 Focus #3)

**Window:** between line 316 (`matter_has_cortex_config` check) and line
359 (`post_to_channel` call). If the cortex-config.md is deleted in this
window, the gate fires for a matter that no longer has a config.

**Cost-estimate read race:** `_read_cost_estimate` (line 330) re-reads the
file. If deleted between 316 and 330, falls back to default ($4). No crash.
The Slack DM still posts. Director approves → `maybe_run_cycle` → Phase 2
has no config to load → garbage cycle.

**Severity:** LOW. The vault is a git-mirrored checkout on Render
(`baker-vault-mirror`). Files don't disappear mid-call in normal
operation — a vault rebase could trigger it transiently (order of
seconds). Even when triggered, the result is one wasted cycle, not a
spend-cap breach. No mitigation needed for this PR.

## §5 — Frontmatter parser edge cases (RA-24 Focus #5)

Tested mentally + via re-reading `_read_cost_estimate` (lines 80-111):

| Input | Parser behavior | Result |
|---|---|---|
| Empty file | `text == ""`; doesn't `startswith("---")` | default ($4) ✅ |
| `---` only, no close | `text.find("\n---", 3)` → -1 | default ✅ |
| Frontmatter without `cost_estimate_dollars:` field | Loop completes, no match | default ✅ |
| `cost_estimate_dollars: "5.0"` (quoted) | `float('"5.0"')` → ValueError | default ✅ |
| `# cost_estimate_dollars: 5.0` (commented) | doesn't `startswith("cost_estimate_dollars:")` | default ✅ |
| Whitespace before key (`  cost_estimate_dollars: 5.0`) | `line.strip()` first → matches | parses correctly ✅ |
| Multiple `cost_estimate_dollars:` lines | Returns FIRST match | acceptable behavior ✅ |
| Tab-indented frontmatter | `line.strip()` handles | ✅ |
| File read I/O error | bare `except Exception` → default + ERROR log | ✅ |
| **`cost_estimate_dollars: nan`** | `float("nan")` succeeds → returns NaN | ⚠️ propagates to Slack DM as "$nan" |
| **`cost_estimate_dollars: inf`** | `float("inf")` → +∞ | ⚠️ "$inf" in DM |
| **`cost_estimate_dollars: -5.0`** | parses → -5.0 | ⚠️ "$-5.00 if approved" in DM |
| **`cost_estimate_dollars: 1e10`** | parses → 10 billion | ⚠️ "$10000000000.00" in DM |

**Observation #2 (LOW, defense-in-depth):** `_read_cost_estimate` accepts
`nan`, `inf`, negative, and arbitrarily large floats. Vault-side
`cortex-config.md` files are trusted (AI Head Tier B writes them per
RA-23) so this is not a security issue. But ~3 lines of bounding —
`if not math.isfinite(cost) or cost <= 0 or cost > 100: cost = DEFAULT_...` —
would prevent embarrassing Slack DMs in case of a typo'd config.

NON-BLOCKING for this PR. Suggest a follow-up one-liner if Wave 1 lands a
related polish brief.
