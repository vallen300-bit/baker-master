## 4. Nice-to-have

### N1 — Strip trailing slash from `$REPO` before sed

`scripts/install_kbl_mac_mini.sh:124`. If Director sets `KBL_REPO=/Users/dimitry/baker-code/` (trailing slash), rendered plist becomes `<string>/Users/dimitry/baker-code//scripts/kbl-*.sh</string>`. macOS resolves `//` correctly so it works, but the result is ugly and visible in `launchctl list` debugging.

**Fix.** One line at the top of the script: `REPO="${REPO%/}"` after the `case` guard. Cosmetic.

### N2 — Refuse-guard could optionally cover iCloud Drive

Current blocklist: `~/Desktop/`, `~/Documents/`, `~/Downloads/`. Task brief asked about `~/Pictures/`, iCloud (`~/Library/Mobile Documents/`).

- `~/Pictures`, `~/Movies`, `~/Music` — unlikely real install locations. Skip.
- `~/Library/Mobile Documents/com~apple~CloudDocs/` (iCloud Drive) — plausible. A tidy operator might keep code repos in iCloud Drive for cross-device sync. Same TCC-class restriction applies in practice. Worth adding one more `case` arm.

The post-sed `grep __REPO__` guard backstops if an unblocked TCC path silently fails — install fails loud either way, just with a less-helpful message. Conservative blocklist is acceptable; extending is strictly better.

**Fix.** One additional `case` arm: `"${HOME}/Library/Mobile Documents/"*)` → fail with same remediation message.

### N3 — Stale `/usr/local/bin/kbl-*.sh` symlinks still resolve to OLD clone

Verified via `ssh macmini`:

```
$ ls -la /usr/local/bin/kbl-*
lrwxr-xr-x  1 root  wheel  62 Apr 18 04:51 /usr/local/bin/kbl-pipeline-tick.sh
    -> /Users/dimitry/Desktop/baker-code/scripts/kbl-pipeline-tick.sh
... (5 symlinks total, all pointing to /Users/dimitry/Desktop/baker-code/scripts/)

$ ls -d /Users/dimitry/Desktop/baker-code
/Users/dimitry/Desktop/baker-code     ← OLD clone still exists on disk
```

B1's report says these symlinks are "harmless because plists don't point there anymore." True for scheduled execution. **But they're not dangling — they actively resolve to the OLD clone**, which still exists at `~/Desktop/baker-code`. Two failure modes:

1. **Manual invocation:** `/usr/local/bin/kbl-pipeline-tick.sh` (e.g., from a debug session) would execute the OLD clone's script, against the OLD clone's repo state, which may have drifted from the new `~/baker-code`.
2. **Two-clones-on-disk drift:** `git pull` in `~/baker-code` doesn't update `~/Desktop/baker-code`. Director (or a future operator) could be confused which is "live."

Neither is a PR #3 issue — the PR doesn't introduce them, it just doesn't clean them up. Per B1's stance, automation would re-introduce sudo. Manual cleanup recommendation:

```bash
sudo rm /usr/local/bin/kbl-*.sh
rm -rf /Users/dimitry/Desktop/baker-code   # confirm 'git pull' is current in ~/baker-code first
```

**Fix.** Optional: add an informational warning at end of installer (`if [ -L /usr/local/bin/kbl-pipeline-tick.sh ]; then echo "INFO: stale symlink at /usr/local/bin/kbl-pipeline-tick.sh — manually rm with sudo if no longer needed"; fi`). Five lines, no sudo, no auto-rm. Could land in a follow-up.

## 6. Specific scrutiny — task asks

### Q1 — `__REPO__` sed substitution safety

```bash
REPO_ESCAPED=$(printf '%s\n' "${REPO}" | sed 's/[\/&]/\\&/g')
sed "s/__REPO__/${REPO_ESCAPED}/g" "${SRC}" > "${DEST}"
```

- **Spaces in path:** quoted everywhere, sed treats spaces literally. ✓ safe.
- **Trailing slash:** results in `//` in rendered plist; macOS-resolved correctly but ugly. See N1.
- **`/` in path:** escaped via `s/[\/&]/\\&/g` pre-pass. ✓ safe.
- **`&` in path:** escaped. ✓ safe (& is sed replacement back-reference).
- **Newlines:** would break, but macOS doesn't allow `\n` in filenames. Not a real risk.
- **`\` in path:** survives the escape pass; could be interpreted as sed back-reference. Edge case for adversarial input, not for normal Director-set paths.
- **Quotes in path:** would break shell parsing. Adversarial only.
- **Placeholder appearing 0 times:** post-sed `grep __REPO__` guard catches this and fails loud. ✓ defensive.
- **Placeholder appearing >1 times:** sed `g` flag replaces all. ✓ safe.

**Verdict.** Robust against normal paths. Adversarial input is a non-concern (Director sets KBL_REPO; not user input). The post-sed grep guard is the right belt-and-suspenders — even if substitution somehow no-ops, install fails loud instead of producing a silently-broken plist. ✓ acceptable.
