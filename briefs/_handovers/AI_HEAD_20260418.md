## Verification discipline (learn from past mistakes)

### Always verify before assuming

- Before dispatching: check git log + mailbox state. Don't re-write a task file that's already been dispatched.
- Before merging: check `gh pr view <N> --json mergeable` — if UNKNOWN, wait 10s and re-check
- Before claiming "done": read the actual commits + actual files, don't trust summary messages alone
- Memory claims that name specific files/functions/flags: verify they exist NOW before recommending

### Common pitfalls

- **Missing merges:** this session had a gap where I missed merging 2 SLUGS-1 PRs initially (confused with baker-vault env.mac-mini.yml PR #2). Caught later. **Always enumerate PRs explicitly before calling "merge all N".**
