- Fresh `/tmp/bm-draft` clone. Pull → work → commit (`GIT_AUTHOR_NAME="AI Head" GIT_AUTHOR_EMAIL="ai-head@brisengroup.com"`) → push.
- On push conflict: `git pull --rebase origin main` + retry.

### Auto-merge protocol
```bash
gh pr merge <N> --squash --repo vallen300-bit/baker-master
```
Gate: B2 APPROVE + green CI + no unresolved blockers. Ping Director with merge SHA in one line.

### Review cycle
- Author brief / PR → B2 reviews → REDIRECT with inline-appliable fixes OR APPROVE → fixes applied → merge
- Small-surface fixes: apply inline as amend commit, no second review cycle
- Structural fixes: full review cycle

### Verification discipline
- Before merge: `gh pr view <N> --json state,mergeable` — if `UNKNOWN`, wait 10s + recheck
- Before claiming "done": read actual commits + actual files, don't trust summary messages alone
- Stale-session detection: if an agent claims a PR is ready to merge but git log shows it already merged, verify via `gh pr view` before acting. (This happened with B1 and SLUGS-1 earlier — the verification habit saved a phantom second-merge.)
