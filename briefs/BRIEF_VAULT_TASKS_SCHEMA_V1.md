## Part 2: Task frontmatter schema

Every task file under `tasks/active/` MUST have this frontmatter. Validation in Part 5.

```yaml
---
title: <one-line human-readable title>
status: queued | in_flight | blocked | done | dropped
due: 2026-12-31                          # ISO date (no time); omit for "no due"
priority: critical | high | normal | low
owner_agent: movie-desk                  # one of the 13 canonical bus slugs OR an agent dir
                                         # under _ops/agents/ (e.g., "ah1", "movie-desk")
matter_slug: mo-vie                      # MUST exist in baker-vault/slugs.yml
created: 2026-05-13                      # ISO date
created_by: ai-head-1                    # agent slug that created the task
references:                              # optional list of paths/URLs
  - wiki/matters/mo-vie/curated/...
blocked_by: []                           # optional list of task filenames blocking this one
                                         # (just the filename, not the full path)
closed_at: null                          # ISO datetime, set on close
outcome: null                            # optional: path to artefact produced
---

## <task title here>

<freeform markdown body — context, sub-steps, agent notes>
```

### Status enum — aligned to cortex-roadmap vocabulary

`queued / in_flight / blocked / done / dropped`. Director-ratified 2026-04-30: NO PARKED.

`blocked` is task-specific (waiting on counterparty, blocked_by another task); not in the roadmap enum but needed here.

### Recurrence + auto_close — NOT v1

Do not add `recurrence` or `auto_close` fields. Recurring jobs live in `schedule-registry.yml` (Part 6), not in per-task frontmatter. Defer to v2.

## Part 3: Closure protocol

When an agent completes a task:

1. Update frontmatter: `status: done`, `closed_at: <ISO datetime UTC>`, `outcome: <path or null>`
2. **Move file** from `tasks/active/<filename>.md` → `tasks/closed/YYYY-MM/<filename>.md` (where YYYY-MM matches `closed_at`)
3. `git add` BOTH the old and new path; one commit:
   ```
   tasks(<desk>): close <task-slug> — <one-line outcome>
   ```

Dropped tasks: same flow, `status: dropped`, move to `closed/YYYY-MM/` (dropped is a closure type, not a separate folder).

Blocked tasks: stay in `tasks/active/`, frontmatter `status: blocked` + `blocked_by: [...]`. Scanner (Brief 2) flags blocked tasks differently in the daily digest.

## Part 5: `slugs.yml` + agent-slug validation lint

Create `scripts/validate_vault_tasks.py` in baker-vault (NOT baker-master — vault is its own repo).

**Behavior:**
- Walk `_ops/agents/*/tasks/active/*.md` (active only; closed/ tasks are frozen audit trail)
- For each file, parse frontmatter and validate:
  1. **Required fields present:** `title`, `status`, `priority`, `owner_agent`, `matter_slug`, `created`, `created_by`
  2. **`status` enum:** must be one of `queued / in_flight / blocked / done / dropped`. (Note: `done` and `dropped` in `tasks/active/` is a state error — flag.)
  3. **`priority` enum:** `critical / high / normal / low`
  4. **`matter_slug` validity:** must exist in `baker-vault/slugs.yml`. Use the existing `kbl/slug_registry.py` loader pattern from baker-master if vault has a mirror, otherwise just parse `slugs.yml` directly with PyYAML.
  5. **`owner_agent` validity:** must be one of (a) the 13 canonical bus slugs (`director / cowork-ah1 / lead / deputy / architect / b1 / b2 / b3 / b4 / b5 / cortex / daemon / aid`) OR (b) a directory name under `_ops/agents/`. Lint is permissive: union of both.
  6. **`due` format:** if present, must be ISO date `YYYY-MM-DD`.
- Exit 0 if all pass, exit 1 on any violation, print per-file diagnostic.

**Lint mode, not block mode:** This is informational v1. Wire into CI later (out of scope for this brief).

Run it once at brief-completion against the MOHG-debrief seed file — must exit 0.
